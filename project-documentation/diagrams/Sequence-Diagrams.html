Document Overview

This document contains two types of sequence diagrams for the Code Quality & Fault Prediction System:

  • System Sequence Diagrams (SSD) - Black box view showing external actor-system interactions
  • Design-Level Sequence Diagrams - White box view showing internal component interactions

Understanding the Difference

White Box View - Internal Design

These diagrams reveal the internal architecture, showing how components interact to fulfill each use case. Lifelines use UML stereotypes:

  • <<boundary>> — UI Components (React Pages)
  • <<control>> — Controllers & Services (Flask Routes, GitHubService, CodeAnalyzer, MLService)
  • <<entity>> — Domain Models (User, Repository, CodeMetric, Prediction, BugReport, Settings)

Design Details:

  • ≪control≫ GitHubService: Uses GitPython (Repo.clone_from) for cloning
  • ≪control≫ CodeAnalyzer: Uses Radon (LOC, complexity, maintainability) + Lizard
  • ≪control≫ MLService: sklearn.ensemble.RandomForestClassifier with StandardScaler
  • Risk Levels: critical (≥0.75), high (≥0.5), medium (≥0.25), low (<0.25)
  • Performance: bulk_save_objects() for batch database inserts
